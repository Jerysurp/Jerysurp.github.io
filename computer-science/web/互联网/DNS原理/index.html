什么是 DNS（Domain Name System）

DNS 就是域名名称系统，是因特网中的一项核心服务，是用于实现域名和 IP 地址相互映射的一个分布式数据库。通过主机名，得到该主机名对应的 IP 地址的过程叫做域名解析（或主机名解析）。DNS 是互联网的基础设施类的服务，这是一张协议，协议最后要实现的软件有很多（如 Bind、coreDNS 等）

DNS 采用 C/S 架构，服务器端工作在 TCP、UDP 协议端口 53

应用程序通常将 DNS 委派给操作系统的 DNS Resolver 来执行，程序员对它几乎没有感知

DNS 域名空间中，树的最大深度不得超过 127 层，树中每个节点最长可以存储 63 个字符

什么是域名（Domain Name）

域名是一串用点分割的字符，是互联网某台或者某组计算机的名称，能够使用户更方便的访问互联网，而不用去记住能够被机器直接读取的 IP 数串

使用域名的好处：

1. 方便记忆

2. 更换机器 IP 地址后不需要改变硬编码在应用的域名，只需要更改解析

3. 便于实现负载均衡，例如一个域名可以对应多个 IP

4. 便于实现虚拟主机名进行访问同一个地址的多个应用，例如 Nginx 的代理

域名结构解析

域名结构是树状结构，树的最顶端代表根域，所有域名的根。根域名服务器负责解析顶级域名，给出顶级域名的 DNS 服务器地址，域名是 “” 空字符串，而 FQDN 是.

根的下一层就是由我们所熟知的.com、.net、.cn 等通用域和.cn、.uk 等国家域组成，称为顶级域，二级域名。每个顶级域名都对应各自的服务器，它们之间是完全独立的。如.cn 的域名解析仅由.cn 顶级域名服务器提供

再下一层就是次级域，三级域名，个人 / 企业能买到的域名，比如 http://baidu.com、http://taobao.com 等等域名，它们基本上是归企业和运维人员管理。每个次级域名都有一到多个权威 DNS 服务器，这些 DNS 服务器会以 NS 记录的形式保存在对应的顶级域名（TLD）服务器中。权威域名服务器则负责给出最终的解析结果

再下一层就是子域，四级域名，它不同于其他层域一样，它是用 * 代替了 www，如 *.baidu.com，而这个域统统都是 baidu.com 的子域。每一个子域都可以有自己独立的权威 DNS 服务器，通过在子域中添加 NS 记录实现。在域名解析时，主机记录输入 * 会提示：泛解析，匹配其他所以域名

三级和四级域名，总体概括来说域名是由整体到局部的机制结构

FQDN

FQDN，完全限定域名，它是使用 DNS 的树状层级结构的完全路径域名来表示一个准确位置对应的主机

域名大致分类如下：

组织域：com、org、mil、gov、edu、net

国家域：cn、hk、tw、us、jp、ir、uk

反向域：in-addr-arpa

什么是名称解析？

名称解析的过程就是某个应用程序

简单来说，它可以将一个域名解析到对应 IP 地址上，从而能访问该 IP 地址下机器提供的相关应用，所以几乎每个 Internet 链接都以 DNS 查找开始

我想大部分都喜欢比较好记的名称来查找计算机，如网络上的服务器。但计算机使用数字地址在网络上通讯。因此为了更容易地使用网络上的资源，DNS 等命名系统提供了一种方法，将计算机或服务的用户用更好记的名称映射为数字地址

DNS 常规解析流程

1、首先客户端位置是一台电脑或手机，在打开浏览器以后，比如输入 http://www.zdns.cn 的域名，它首先是由浏览器发起一个 DNS 解析请求，如果本地缓存服务器中找不到结果，则首先会向根服务器查询，根服务器里面记录的都是各个顶级域所在的服务器的位置，当向根请求 http://www.zdns.cn 的时候，根服务器就会返回.cn 服务器的位置信息

2、递归服务器拿到.cn 的权威服务器地址以后，就会寻问 cn 的权威服务器，知不知道 http://www.zdns.cn 的位置。这个时候 cn 权威服务器查找并返回 http://zdns.cn 服务器的地址

3、继续向 http://zdns.cn 的权威服务器去查询这个地址，由 http://zdns.cn 的服务器给出了地址：202.173.11.10

4、最终才能进行 http 的链接，顺利访问网站

这里补充说明，一旦递归服务器拿到解析记录以后，就会在本地进行缓存，如果下次客户端再请求本地的递归域名服务器相同域名的时候，就不会再这样一层一层查了，因为本地服务器里面已经有缓存了，这个时候就直接把 http://www.zdns.cn 的 A 记录返回给客户端就可以了，但这个缓存一般只有两天

DNS RR 资源记录

DNS 层级结构中不管是节点还是叶子节点都是资源，对这些资源中的某一个的标识使用一定格式的多字段的一条记录来标识，该条记录就是资源记录 RR

当我们查询域名 http://www.zdns.cn 的时候，查询结果得到的资源记录结构体中有如下数据：

1、TTL，生存周期，是递归服务器会在缓存中保存该资源记录的时长

2、class，网络 / 协议类型，表示一个协议或者一族协议，常用 IN 表示 internet

3、type，资源记录类型，一般的网站都是都是 A 记录（IPv4 的主机地址）

4、rdata，资源记录数据，就是域名关联的信息数据

DNS 服务器的分类

常见的 DNS 服务器就是两种：权威解析服务器和递归解析服务器。递归解析服务器也可以叫做 localDNS

权威解析服务器

DNS 权威服务器保存着域名空间中部分区域的数据。如果 DNS 服务器负责管辖一个或多个区域时，称此 DNS 服务器为这些区域的权威服务器

根权威 DNS 或者二级权威服务器中的资源记录标记被指定为区域权威服务器的 DNS 服务器。通过资源记录中列出服务器，其他服务器就认为它是该区域的权威服务器。这意味着在 NS 源记录中指定的任何服务器都被其他服务器当作权威的来源，并且能肯定应答区域内所含名称的查询

递归服务器

递归服务器在正常情况下，初始的时候里面没有任何域名解析数据，里面所有的域名解析数据都来自于它到权威解析服务器的查询结果，一旦查询完毕，递归服务器就会根据 TTL 时间在本地形成一条缓存记录，并为用户提供 DNS 解析的查询服务，这是递归服务器的功能

小知识：

常用的资源记录类型：

A ：Address 地址，由域名解析出对应 IPv4 地址，属于最基本的记录

AAAA：Address 地址，IPv6 可以理解为新一代 A 记录，以后会逐步提高使用

NS ：Name Server，名称服务器解析记录，是一个 Bind Server，这个 Server 不能提供这个域名的解析服务，直接返回 NS 记录，用户用这条记录请求权威的解析 Bind Server，返回解析地址

SOA：Start of Authority，区域授权起始记录，区域文件第一条记录，而且一个区域文件只能有一条

MX：Mail Exchanger，邮件服务器解析记录，配合 A 记录

CNAME：Canonical Name，别名记录，记录域名与另一个域名的对应关系，由别的域名提供 A 记录的解析

PTR：Pointer，逆向解析记录（IP > 域名），基本只在设置邮件服务器时用到

TXT：Text，文本记录主要用于验证域名所属权以及证书申请校验时使用

SRV：提供服务发现，和 service 有关，主要用于企业域控（AD）、微服务发现（Kubernetes）等

当我们修改 DNS 解析服务记录的时候，我们应该在权威里面修改还是在递归里面修改？所有的 DNS 解析记录里面类型的修改都是在权威解析服务器里面做修改

全球有 13 组 DNS 根服务器、11 台在美国，如果你要问为什么那么多，只能说因为世界上第一台计算机出自美国、一组在挪威、一组在日本

目前国际 DNS 系统中已有上千个 TLD，包括中文甚至藏文域名，除了国际可用的 TLD 还有一类，最常见的比如 ， 被广泛用在集群通信中

一些基础术语

主 DNS 服务器：一台存储着原始资料的 DNS 服务器

从 DNS 服务器：使用自动更新方式从主 DNS 服务器同步数据的 DNS 服务器，也叫辅助 DNS 服务器

缓存服务器：不负责本地解析，采用递归方式转发客户机查询请求，并返回结果给客户机的 DNS 服务器。同事缓存查询回来的结果，也叫递归服务器

转发器：这台 DNS 发现非本机负责的查询请求时，不再向根域发起请求，而是直接转发给指定的一台或者多台服务器。自身不带缓存查询结果

hiht：根 DNS internet 服务器集

递归查询：客户机和本地 DNS 服务器直接查询方式

迭代查询：本地 DNS 服务器多次请求重复查询方式

WHOIS：域名数据库查询，一个域名所有者可以通过查询 WHOIS 数据库而被找到；大多数根域名服务器基本的 WHOIS 由 ICANN 维护，而 WHOIS 的细节则由控制那个域的域注册机构维护。对于 240 多个国家代码顶级域名（ccTLDs）通常由该域名权威注册机构负责维护 WHOIS，如中国互联网络信息中心负责.CN 域名的维护

公共 DNS 服务器：缓存了大量的 DNS 记录，有效降低了上游 DNS 服务器的压力，也加快了网络上的 DNS 查询速度。如谷歌的 8.8.8.8,、腾讯的 114.114.114.114,、阿里的 233.6.6.6 等

DNS 泛解析通配符 * ：DNS 记录允许使用通配符 * ，该通配符可以匹配任意级数的子域，但无法匹配自身

TTL：为了精确控制 DNS 记录的过期时间，每条 DNS 记录都要求自定义设置一个时间属性，单位为秒。任何一条 DNS 缓存，在超时过期时间后都必须丢弃，在没超时的时候，DNS 缓存也可以被主动或者被动地刷新

本地 DNS 服务器：它只在当前局域网内有效，企业常通过 DHCP 或者手动配置方式使内网的服务器都默认使用局域网 DNS 服务器进行解析，该服务器可以只解析自己的私有 DNS 域，而将其他 DNS 域的解析 forward 到公网 DNS 解析器去

私有 DNS 域：它会覆盖掉公网的同名域（前提是公网上有），它也可以使用公网不存在的 TLD，如 xxx.local xxx.lan 等
