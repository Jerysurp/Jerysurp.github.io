# 数据类型

# 数值型 (number)

不区别整型数值和浮点型，所有数字都是浮点型。采用 IEEE754 标准定义 64 位浮点格式表示数字。数字直接出现称为数值直接量 (Numeric Literal)。数值前加符号构成负数，是一元求反运算符，不是此语法一部分。

# 整型数据

十进制整数是一个数字序列

# 十六进制和八进制

十六进制以 0X 和 0x 开头，其后跟随十六进制数字串直接量。

十六进制数字 0-9 和 a (A)~f (F)，表示 0-15 某值

ECMAScript 标准不支持八进制数据，js 某些实现却允许采用八进制（基数 8）

八进制数字 0 开头，后随数字序列，0-7 之间

# 浮点型数据

具有小数点，采用传统科学记数法。一个实数值可被表示整数部分后加小数点和小数部分

可用指数法表示浮点型，实数后跟 e 或 E，后加正负号，其后再加整型制数。记数法表示数值等于前面的实数乘以 10 指数次幂

3.12e11 //3.12*10 的 11 次方

1.234E - 12 //1.234*10 的 - 12 次方

# 字符串型（String）

Unicode 字符、数字、标点等组成序列，表示文本数据类型

包含在 ' ' 单引号 " " 双引号中，双引号内出现双引号或单引号内出现单引号要转义

# Function () 构造函数与函数直接量

构造函数允许运行时动态创建和编译 js 代码，函数直接量却是程序结构静态部分，像函数语句一样

每次调用构造函数都会解析函数体并创建一个新的函数对象。对构造函数调用出现在循环中或一个经常被调用的函数中，效率将非常低。函数直接量出现在循环体还是嵌套函数中，既不会每次调用被重新编译，也不会每次遇到都创建新的函数对象

构造函数创建的函数使用的不是静态作用域。该函数总被当作顶级函数编译

# W3C 引用

W3C 模型 event 对象引用复杂。多数必须明确 event 对象作为参数传递到事件处理函数中。event 对象也可自动作为参数传递，取决事件处理函数与对象绑定方式

onkeyup="example(event)"

唯一可像全局引用一样明确引用 event 方式。只作事件处理函数参数。多个参数可以任意顺序排列
